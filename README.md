# PR-Service

## Структура проекта 
- `/api` - `openapi.yaml` из условия
- `/docs` - изначальное условие задания
- `/deployment` - всё для деплоя приложения из под докера
- `/migrations` - миграции
- `/configs` - тут лежит `config.yaml` (про конфигурацию ниже)
- `/cmd` - точка входа
- `/pkg`
    - `/config` - пакет в котором описаны правила получения конфигураций 
    - `/db` - пакет в котором подключение к бд из конфига
    - `/logger` - пакет в котором реализован логгер, зависящий от конфига
    - `/server` - пакет в котором создается http сервер из конфига
- `/internal`
    - `/models` - все сущьности
    - `/repositories` - слой для работы с бд (postgres)
    - `/services` - сервисный слой
    - `/transport` - слой со всеми хендлерами

## Конфигурация 
Половина данных конфигурации идет из `.env` (обязательные) вторая половина из `configs/config.yaml`.

Пример `.env` можете найти в файле `.env.example`.

Пример `config.yaml`:
```yaml
logging:
  level: debug # debug | info | warn | error
  rotation:
    name: "./logs/app.log" # path/to/file где будут храниться логи
    max_size: 1 # максимальный объем в МБ файла  
    max_backups: 1 # максимальное значение бекапов, если 0 - неограниченно
    max_age: 1 # длительность жизни файла в днях
    duplicate_to_console: true # если включена ротация то можно дублировать в консоль логи
database: 
  max_connections: 10 # макс кол-во подключений к бд (обязательное поле)
```
